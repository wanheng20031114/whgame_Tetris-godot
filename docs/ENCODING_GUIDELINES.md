# 编码规范（基础）

更新时间：2026-04-04

## 1. 统一编码

- 所有文本文件统一使用 `UTF-8`。
- 不使用 `GBK/GB2312/ANSI`。
- 新文件默认 `UTF-8`。

## 2. 行尾与空白

- 统一使用 `LF` 行尾。
- 文件末尾保留一个换行。
- 避免混用全角空格与不可见字符。

## 3. 适用文件

- `.gd`、`.tscn`、`.tres`
- `.md`、`.json`、`.js`
- `.csv`（多语言资源）

## 4. 协作约定

- 提交前如果出现中文乱码，先检查文件编码再改内容。
- 不在同一次提交中混入“大量格式化 + 逻辑改动”，优先拆分提交。
- 历史乱码文件优先在独立提交中转换编码，降低回归风险。

## 5. 本项目建议

- Godot 脚本与场景统一 `UTF-8`。
- Node 服务端与文档统一 `UTF-8`。
- 编辑器设置中固定默认编码为 `UTF-8`。
