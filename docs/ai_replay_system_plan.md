# 俄罗斯方块：AI 辅助复盘与打分系统更新计划

## 1. 概述 (Overview)
本计划旨在为现有的俄罗斯方块项目引入**对局复盘系统**与**AI 走步打分功能**。
通过扩展现有的游戏数据采集逻辑，玩家将能够在游戏大厅中浏览历史对局（Demo/Session JSON）。进入复盘模式后，可以逐帧回放每一步的操作，并在可视化界面和时间线中，清晰地看到 MLP 模型对玩家每次落子的评价与打分。

## 2. 核心机制与数据流向

### 2.1 游戏数据的定义与扩充
对局中产生的每次落子都被视为复盘时间线上的一帧。现有的 `session`（JSON 数据）相当于一个游戏 Demo 录像。
为支撑 MLP 模型的推理与复盘的高精度展示，我们需要大幅增加采集瞬间的数据量。

**关键数据指标 (MLP 必须输入的 4 个状态特征)**：
1. `lines_cleared`: 该步操作带来的消除行数。
2. `holes`: 落子并**消行后**，棋盘上的空洞总数。
3. `bumpiness`: 落子并**消行后**，各列高度差的总和。
4. `total_height`: 落子并**消行后**，所有列高度的总和。

**数据采集的时机与方式**：
- 在方块自然下落或硬降锁定（Lock）并触发消行结算的瞬间进行采集。
- 所有的地形数据（Holes, Bumpiness, Total Height）计算逻辑将集中实现在 `PlayerDataCollector.gd` 中。
- 在 `record_piece_drop` 接口中，记录并保存两个不同状态的棋盘：
  1. `board_state_after_drop`：下落瞬间但**未消行**的棋盘二维数组。
  2. `board_state_after_clear`：下落且**消行后**的棋盘二维数组（用于底层结构评估与下一次渲染过渡）。

### 2.2 AI 评估与持久化逻辑
复盘过程中的 AI 打分并非在游戏中实时计算，而是采用“按需分析与持久化”的桥接方式：
1. 玩家在大厅点击并选择某个具体的 Demo JSON。
2. 游戏检测本地是否存在该局的分析文件（例如命名规则为 `session_ID_analyzed.json`）。
3. **若不存在**：则触发外部程序，调用内置的 MLP 计算模型（Python 脚本）。该程序遍历原始 Demo 中记录的每一步地形数据，生成相应的评价分数。计算完成后，将分数打包生成新的 `_analyzed.json` 进行持久化存储。
4. **若已存在**：直接读取，进入复盘场景。

> *注：后续如果需要实现“AI 落点建议（幽灵方块预判）”，也将在分析阶段一并生成建议坐标并存入 JSON，当前阶段仅实现**打分评价**。*

## 3. 详细实施步骤 (Implementation Steps)

### 3.1 核心数据采集扩展
**修改目标**：`scripts/core/player_data_collector.gd` 与核心棋盘逻辑。
- 增加内部计算方法 `_calculate_holes()`, `_calculate_bumpiness()`, `_calculate_total_height()`。
- 修改 `record_piece_drop`，使之能接收并存储未消行和消行后的两份棋盘数据。
- 在 `snapshot` 数据字典中增加这些衍生数据（同时保留 `piece_type`, `rotation`, `col`, `row`, `elapsed_since_last_piece_ms` 等）。

### 3.2 UI 与系统入口扩展
**修改目标**：`scripts/ui/main_menu.gd` 及大厅相关 UI。
- 在大厅主界面中，新增一张“对局复盘 (Replays/Demo)”的模式入口卡片（与现有的“马拉松”等卡片对齐）。
- 点击卡片后，弹出文件列表视图，解析 `userdata/sessions/` 目录下的对局记录供玩家选择。

### 3.3 复盘模式场景构建
**修改目标**：新增 `scenes/ui/replay_system.tscn` 及其关联脚本。

复盘场景将被设计为三栏或多区域交互界面：
1. **中央核心视区 (Game View)**：
   - 包含标准的 10x20 棋盘、Next 队列视窗和 Hold 视窗。
   - 画面渲染的总是 `board_state_after_drop`（玩家刚落下，还未消除的瞬间）。
   - **高亮系统**：由于 JSON 中已保存 `piece_type`, `rotation`, `col`, `row`，无需额外抓取画面坐标。在渲染当前帧时，直接反向推演该方块占据的格子数组，在棋盘上赋予特殊的高亮颜色或描边。
   
2. **侧边时间线 (Timeline Slider/List)**：
   - 一个可上下滑动的时间轴，每个单元格代表一次落块。
   - 单元格内显示：该次下落的方块图标、步数序号、以及该步的 **AI 模型打分**。
   - 玩家点击任意单元格，中央视区与数据面板立即跳至该状态。
   
3. **侧边数据面板 (Data Panel)**：
   - 展示玩家下落这一刻的所有深层指标，包含：
     - 当前采用的消除类型（普通、T-Spin、B2B）与消除行数。
     - 各种消除类型的使用比例统计。
     - 本次下落的用时（`elapsed_since_last_piece_ms`）。
     - MLP 输入矩阵四维数据（Lines, Holes, Bumpiness, Height）。

## 4. 总结
通过本次系统升级，玩家不仅能以回放录像的方式欣赏自己的操作，更能通过时间线直观对比出哪一步操作是“神仙手”，哪一步导致了“堆叠崩盘”。将底层计算与 `PlayerDataCollector` 的分离，保证了游戏本体在录制时的高性能；同时采用持久化 AI JSON 的策略，确保了回放过程的零延迟与可共享性。
