# SRS (Super Rotation System) 常量矩阵与偏移表

## 1. 基础矩阵体系 (Base Shapes)
现代俄罗斯方块的灵魂所在——旋转原点不再位于世界左上角，所有方块通过 3x3 或 4x4 的边界盒旋转：
*   **O 块 (2x2)**：实际不会产生位移偏移
*   **I 块 (4x4)**：最长直线
*   **J, L, S, T, Z 块 (3x3)**：主流通用旋转体系

> [!NOTE]
> 这里的数组遵循 [y][x] 或基于笛卡尔坐标系的矢量定义。

## 2. 状态机制
共有 4 个朝向：
*   **0**: 初始状态 (出生时面向玩家)
*   **R (1)**: 顺时针旋转 90 度
*   **2**: 倒置 (180 度)
*   **L (3)**: 逆时针旋转 90 度

因此所有墙踢偏移数组均分为 8 个检测场景 (0->R, R->0, R->2, 2->R, 2->L, L->2, L->0, 0->L)。

## 3. 墙踢表 (Wall Kick Offset Data)

所有墙踢测试最多进行 **5次尝试 (Test 1 ~ 5)**。每一次尝试代表一个相对当前 (x,y) 的向量偏移 `(Δx, Δy)`。在 Godot 中，Y轴向下为正，因此以下数据在导入到我们的代码(`scripts/core/piece_data.gd`)时需要基于 Y轴方向 做好反转映射（以下依据标准国际基准坐标，Y 向上为正）：
